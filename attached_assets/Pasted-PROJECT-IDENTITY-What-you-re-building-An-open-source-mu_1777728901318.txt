CORE ARCHITECTURE
System Components
1. Agent Orchestrator (Python + asyncio)

Lightweight coordinator managing agent lifecycle and communication
Uses Redis for agent state and task queue (optional: in-memory for simplicity)
Implements circuit breaker pattern for LLM API failures
Total: ~500 lines of clean Python

2. Four Specialized Agents
Agent A: Test Generator

Input: Source code file or function
Output: pytest test cases with edge cases
Model: Claude 3.5 Sonnet (via Anthropic API) or local Qwen2.5-Coder-7B
Instrumentation: logs test coverage metrics, generation time, token usage

Agent B: Log Analyzer

Input: Test execution logs, stack traces
Output: Structured failure analysis (root cause, affected components, severity)
Model: GPT-4o-mini (cost-effective) or local Llama-3.3-70B
Instrumentation: tracks pattern matching accuracy, false positive rate

Agent C: Failure Triage Agent

Input: Failure analysis from Agent B
Output: Priority ranking, duplicate detection, regression classification
Model: Claude 3.5 Haiku (fast + cheap) or Gemini 2.0 Flash
Instrumentation: measures triage accuracy against human labels

Agent D: Bug Reporter

Input: Triaged failures
Output: GitHub issue via API (title, description, labels, assignment)
Model: GPT-4o-mini (formatting + API calls)
Integrations: GitHub API, Jira API, Linear API via MCP

3. Evaluation System

Golden test set: 50 curated examples across 5 failure types
Metrics: precision/recall for triage, test coverage %, false alarm rate
Automated eval runs on every pipeline change
Results logged to SQLite + visualization dashboard

4. Observability Layer

Structured logging (JSON lines) to stdout + file
OpenTelemetry traces for agent communication
Prometheus metrics: agent latency, LLM token usage, failure rates
Grafana dashboard (optional): visualize pipeline health
